# ⚾ 구속 측정기 (Baseball Speed Gun)

iPhone으로 찍은 영상을 분석해 투구 속도를 측정하는 **서버 없는 정적 웹앱**입니다.
`index.html` 파일 하나가 전부이며, 모든 분석은 브라우저 안에서 처리됩니다.
캐치볼·동네야구(~100km/h)가 주 대상이며, 정식 야구장 고속 투구도 슬로모션으로 지원합니다.

## 원리

1. 옆에서(공이 화면을 가로지르게) 투구를 촬영
2. 웹앱에서 영상을 불러와 프레임 단위로 탐색
3. 화면에서 실제 길이를 아는 두 점을 탭해 **픽셀↔미터 환산 기준** 설정
4. 릴리스 직후 프레임의 공(A)과 몇 프레임 뒤의 공(B)을 탭
5. 속도 = (픽셀 거리 ÷ 환산 비율) ÷ (프레임 수 ÷ 촬영 fps)

## 어떤 영상으로 찍나

- **~100km/h (캐치볼, 동네야구)**: 일반 영상이면 충분. **60fps 권장**(설정 → 카메라 → 비디오 녹화 → 1080p/60fps).
  앱에서 촬영 프레임레이트를 "자동"으로 두면 끝.
- **100km/h 이상 (정식 투구)**: 240fps 슬로모션 권장. 이때는 앱에서 촬영 fps를 240으로 직접 선택해야 합니다.
  슬로모션 파일은 재생 타임라인(슬로우 구간 ~30fps)과 실제 촬영 fps가 달라서 자동 감지가 불가능하기 때문입니다.

## 촬영 팁 (정확도의 대부분이 여기서 결정됩니다)

- **측면에서 촬영**: 투구 궤적과 카메라 시선이 직각이 되게. 뒤/앞에서 찍으면 측정 불가.
  던지는 라인 옆으로 8~10m 떨어지면 10m 남짓의 투구 전 구간이 화면에 들어옵니다.
- **삼각대 또는 고정**: 흔들리면 캘리브레이션이 틀어집니다. 벤치나 가방 위에 올려도 됩니다.
- **거리 기준 만들기 (동네 환경)**: 던지는 사람과 받는 사람의 발 위치 간 거리를 줄자나 걸음으로 재두는 게 가장 좋습니다.
  대안: 5m 간격으로 놓은 물병 2개, 라인 옆에 서 있는 친구의 키.
- 기준선의 두 점은 **공의 비행 경로와 비슷한 깊이**에 있어야 합니다.
  카메라에서 가까운 물체를 기준으로 잡고 먼 곳의 공을 재면 과대/과소 측정됩니다.
- A–B 마킹 간격은 **10프레임 이상** 권장 (간격이 짧을수록 탭 오차의 영향이 커짐).

## 실행 / 배포

서버가 필요 없는 정적 파일입니다.

- **PC에서 열기**: `index.html`을 브라우저로 열면 끝.
- **같은 Wi-Fi의 iPhone에서 테스트**:
  ```
  npx -y serve -l 5173 .
  ```
  실행 후 iPhone Safari에서 `http://<PC의 IP>:5173` 접속.
- **언제든 쓰려면 (권장)**: GitHub Pages / Netlify / Cloudflare Pages 등에 `index.html`을 올리고,
  iPhone Safari에서 열어 **공유 → 홈 화면에 추가**하면 앱처럼 사용할 수 있습니다.

## 정확도에 대해

- 이 방식은 A–B 구간의 **평균 속도**를 측정합니다. 스피드건(릴리스 최고 구속)보다 약간 낮게 나오는 것이 정상이며,
  A를 릴리스 직후에 가깝게 잡을수록 스피드건 값에 근접합니다.
- 조건이 좋으면(측면 촬영 + 정확한 캘리브레이션 + 240fps) ±2~5km/h 수준을 기대할 수 있습니다.

## 테스트 영상

속도를 정확히 알고 만든 합성 영상으로 앱을 검증할 수 있습니다 (ffmpeg 필요):

```powershell
node tools\make-test-video.mjs 70 test-normal.mp4 60 60 10      # 일반 60fps, 기준 10m, 70km/h
node tools\make-test-video.mjs 140 test-slomo.mp4 240 30 18.44  # 슬로모션 구조, 140km/h
```

영상 속 두 기둥 밑동(빨간 점) 사이가 기준 거리입니다. 인자: 속도, 출력파일, 촬영fps, 재생fps, 기준거리(m).
